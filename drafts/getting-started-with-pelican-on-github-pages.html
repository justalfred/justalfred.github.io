Getting Started with Pelican on Github Pages

Date Fri 02 January 2015 Tags pelican / publishing

I want more people to blog. I want people to give voice to their experiences. Especially those people who think they have nothing to say. It's not that I think we need more noise on the internet. It's that the conversations out there are often dominated by a few loud voices that are often not representative of the masses. If

There are various guides out there that show how easy it is to get started publishing with pelican. My problem is that I don't like uncertainty. What are all these auto-generated files for? How do I customize the look and organization? How can I be sure I don't expose more information than I want? Because of my insecurity, it took me a lot longer to get started than I wanted. Here is my summary of the steps I ended up taking, and an explanation of what each step does. Hopefully this will save the reader time bouncing around various tutorials, blog posts, and documentation.

Intent

I've only just started using pelican, so I can only speak to what I've done so far. I wanted a blog which allows me to draft in markdown. I didn't want to pay much money, and I wanted to avoid having to serve a website. After a few months of indecision, I settled on using pelican to generate a static website which I would put on Github Pages. Pelican has the advantage of being well supported and in a language that I like (Python). Github Pages is free and allows me to use a custom domain. I guess that's just a vanity feature.

Install

So let's get started. You can create a new python virtual environment if you want. If you're sloppy and lazy like me, just go ahead and install a few packages.

pip install pelican markdown typogrify
